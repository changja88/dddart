## B. dddart 맥락 재구성 — dddjango에 없는 신규 절

- **§5 판정 소유와 강등** (규약 §3.3 — dddart 고유 1급 규칙): 도메인 어휘 판정은 1곳째부터 domain 기본·VM 소유 주장엔 *왜*·Model 밖 2곳 복제 시 domain_service/specification 강등. 운반자: 본설계 §5-2(명세 라벨링)·§6-3(discipline-reviewer 판정 소유 대조)·undecidable §8. state final §2가 전방 위임해 둔 주제(백필 대상).
- **§4 애그리거트 클라 번역** — §10-5 ③ 결정 반영(아래 C).

## C. §10-5 ③ 결정 지점 (사용자 확인 — 합성 중 단일 미결)

규약 문면: "애그리거트 '일관성 경계'의 코드 규율(루트 경유 변경 원칙 등 — freezed 불변+직파싱 하에서의 최소 규칙, VM 판정 강등 규칙(§3.3)의 상세 포함)". 후보 3안과 권장은 본 리뷰 시점에 사용자 제시 — 결정 후 이 절에 기록한다.

**[결정 — 2026-06-12 사용자 확정: 안 A]** 3규칙:
1. **루트 경유 변경**: 조건·계산·상태 전이가 걸린 갱신은 애그리거트 루트의 메서드가 새 인스턴스를 반환하는 형태로만(`order.cancel()` → 새 Order). VM·UseCase의 `copyWith` 직접 호출은 도메인 의미 없는 단순 복제에만 허용 — copyWith는 누구나 부를 수 있는 만능 문이라 freezed 불변만으로는 규칙 산개를 못 막는다.
2. **불변식은 변경 메서드 안에서 검증**: 전이 조건 위반은 도메인 예외(`exception.dart`) throw. 생성 시점 검증은 강제하지 않는다 — 직파싱 전제에서 생성자 검증은 서버 데이터가 들어오는 길을 막는다. 클라가 책임질 것은 사용자가 일으키는 변경뿐.
3. **타 애그리거트 참조**: 서버 응답이 중첩이면 중첩 그대로(직파싱 정합), 클라가 새로 조립하는 관계만 ID 참조 우선. Vernon 규칙 3의 ORM FK 확장은 서버 관심사라 제거.
규약 개정 불요(§10-5 ③ 문면 "루트 경유 변경 원칙 등 — 최소 규칙"의 직역). ddd final §4에 본문 반영, 판정 소유(§5)와 한 쌍.

## D. 경계 결정

- vs **houserules**: domain_layer 트리·5종 폴더·명명 표는 houserules §1·§4 위임. ddd는 종류별 판별·작성 규율.
- vs **state**: VM의 일(번역·변환)은 state §2 — 판정 소유 본문은 ddd 소유(state §2의 전방 위임 백필). 애그리거트 간 화면 동기화는 state §5(SharedState) 위임.
- vs **data**: Repo 구체·Either·직반환은 data §1·§3·§4 위임. ddd §10 비채택(Repo 인터페이스·Data Mapper)에서 data 위임 인용.
- vs **cleancode(동시 작성)**: 캡슐화·SOLID·이름 일반론은 cleancode — ddd는 도메인 모델 특정 규율만. 빈혈(§9)은 ddd 소유(도메인 모델 어휘), cleancode §6 캡슐화에서 ddd 위임.
- **undecidable §3·§4·§8**: BC 어휘·게이트·판정 귀속 판별 절차는 공유 reference 소유 — ddd final은 위임+규칙 본문.

## E. 공백·발명 위험 (P1 기록 대상)

1. Dart 예제 — external의 Python 예제를 freezed/Dart로 치환하되, **§10-5 ③ 확정 규칙의 예제는 결정 후 작성**(미결 상태에서 코드 형태 선취 금지).
2. Vernon 규칙 1(트랜잭션)·4(결과적 일관성)의 클라 번역 — 원료(규약)에 명문 없음: "클라엔 트랜잭션 없음·서버가 진실원천·화면 동기화는 SharedState/재조회"는 규약 §9-15(이벤트 제거 근거)·state §5에서 도출 가능한 조립. P1 표에 도출 근거 기록.
3. 강등 판별 상세(spec vs domain_service 선택 기준) — 규약 §3.2 문면("재사용·조합되는 판정 규칙"=spec)에서 직접 도출.
4. **§7 조합 계층 지연 규칙**(4렌즈 1라운드 사후 기록): "AndSpecification 류는 조합 수요가 실재할 때만"은 원전(전체 계층 표준 제시)에 없는 dddart 선별 결정이다 — 골격 과잉 방지(규약 §5 "코드는 필요할 때만" 정신)로 채택·유지. 결정 주체: 합성(2026-06-12), 이 항목이 그 기록.
5. **§8 발명 제거**(fidelity 렌즈 P1 교정): UseCase 예제의 도메인 예외 처분 주석("삼키지 않는다·버그 표면화")은 원료 무근거 발명이라 제거. **도메인 예외(전이 위반)의 최종 처분 경로(누가 잡아 사용자에게 어떻게 보이나)는 코퍼스 미규정** — 사전 판정(canX getter) 소비로 UI가 진입을 차단하는 것이 자연 경로이나 명문 규칙은 없다. 실전 발생 시 결정 후보(사용자 보고 완료 — Wave 3 보고).
6. **§10 보강**(fidelity 렌즈 P2 교정): 대규모 구조·마이크로서비스식 BC 분리 2행을 비채택 표에 추가(review A의 행선 기록과 표 실물의 누락 해소).
