# 이미지 충실도 — 생성측 유도 + 육안 판정 (작업 B 설계·2026-06-22)

> **목적**: 이미지 충실도(위치·존재)를 *측정*으로 자동화하려는 시도를 접고, **생성측(§9)이 제자리에 그리도록 유도 + 사용자 육안이 판정**하는 구조로 정리한다. 그 함의로 **FID 게이트에서 image를 제거**한다(작업 C로 활성화된 거짓 FAIL 차단).
> **트리거**: 작업 B(image set-membership)가 적대 검증에서 4결함(area↔slot 비대칭·dedup·L2 은폐·repeat 비대칭) 노출 → 분리 → 사용자 방향 전환("측정으로 위치 못 잡음 → 생성측+육안") → 3렌즈 정합성 검증 → 설계 문서 적대 정합성 리뷰(생성측 보정).
> **사용자 확정(2026-06-22)**: "image 측정은 하지 마, 내가 육안으로 본다." 위치·존재 전부 육안. 약신호(0개 ⚠)도 안 함.
> **제약**: feedback-015 공리(**코퍼스 레이아웃 어휘 0·명세에 형상/위치 기입 금지**)·결정론·양판 미러·측정 도구 positive-control 재검 선결·다음 런 동결·별도 승인.

## 1. 근거 (3렌즈 정합성 + 설계 적대 리뷰 결과)

- **측정으로 위치 자동화 불가**(적대 검증): image가 layout-ir에서 area(top-level)·slot(컨테이너 내부)로 *레벨 분산*되고, 두 파서(extract/dump)의 dedup·repeat 추상화가 비대칭 → L1·L2 어디서 비교해도 샌다. "총수 비교"는 4결함을 낳는다.
- **★작업 C로 모순 활성화**(렌즈 2·실측): 작업 C(screenProbes)가 FID 게이트를 켰으므로, 현재 `compare_layout`이 image를 area role로 보는 로직이 **15차부터 거짓 FAIL을 실현**한다(이미지를 섹션 안에 합리적으로 두면 `누락=[image]`·`추가=[image]`로 L1·L2 동시 ❌·exit 2 실측). "위치=생성측 자유(§9) ↔ 측정=위치 강제(L1)"의 모순이 *지금 막* 활성화됐다.
- **위치는 본질적으로 육안**(feedback-015 오라클): 시안 100% 재현(사용자 공리)에서 *위치·미세 충실도*는 사람 눈이 판정한다. 13·14차 codex 이미지 문제를 잡은 것도 도구가 아니라 사용자 육안.
- **생성측은 이미 image 위치를 §9에 귀속 중**(설계 적대 리뷰 정정): coder.md `:15`("레이아웃 형상은 예외 — design-ref 근거 §9")·`:24`("design-ref = 형상 단일 근거·배치·축 충실 재현 §9·시각 근거에 그치지 않는다")·`:40`("형상은 design-ref에서 가져와 재현")이 **이미 *모든 요소*(image 포함) 배치를 §9에 귀속**시킨다. §9 본문(`:243`)도 "빠짐없이 재현". → **"빈틈"은 없다**(직전 초안의 "coder:25→§9 연결 빈틈"은 과장·오진단으로 정정). 유일한 미세 공백 = §9가 형상 어휘를 텍스트 요소 중심으로 예시해 `<img>`를 *콕 집지 않음*.
- **3축 이미 분담**(렌즈 3): 존재/정체=§8 에셋·위치=§9 형상·크기=architecture-ui §8(size-link)가 코퍼스에서 서로 명시적 카브아웃. codex 14차 "통째 안 그림"=에셋 공급 실패(fetch 0건)로 확정·위치 무관·별도 라운드.

## 3. 작업 1 — 생성측: §9에 "`<img>`도 형상" 한 구절 (최소·명세 무기입)

**상태**(설계 적대 리뷰): coder.md `:15·:24·:40`이 이미 모든 형상(image 포함)을 §9에 귀속한다 — **빈틈 없음**. §9 본문도 "빠짐없이 재현". 유일한 미세 공백 = §9 형상 예시가 텍스트 요소 중심이라 `<img>`가 형상 재현에서 *암묵적으로* 다뤄질 뿐 콕 집히지 않음.

**처방**(단 한 곳·시안 재현 의무로·명세 무기입):
- **implementation-flutter §9에 "`<img>`(이미지)의 컨테이너 내 위치·형제 순서도 형상의 일부 — 시안 그 자리에 재현"을 명시.** 단 인용·삽입을 정확히:
  - `:245`의 "시안의 모든 배치 단서를 충실히 옮긴다"는 **볼드 없는 평문이고 `(예: flex-col→Column …)` 괄호 예시 *내부*** — 그 괄호 안에 또 괄호를 넣지 않는다.
  - 삽입은 **괄호 밖**: §9:243 본문("**빠짐없이** … 재현") 직후 또는 §9 별도 불릿으로 "`<img>`도 형상"을 한 구절. *별도 강제 절·닫힌 매핑 표는 만들지 않음*(feedback-015 공리 — "닫힌 목록 아님" 유지). 정확한 위치/문구는 plan에서 실파일 보고 확정.
- **coder.md 불변**(`:15·:24·:40`이 이미 §9 귀속·중복 회피). architect·review-ui 불변(위치를 명세에 안 적음·공리 준수).

**양판 미러**: §9 final.md `--write`(배포→소스+codex 3사본). **coder.md 미변경이므로 수동 codex 불요.**

**검증**: 앵커 grep(§9 `<img>` 구절·괄호 밖 확인)·3사본 대칭. *효과(coder가 제자리에 그림)는 육안·다음 런*.

## 4. 작업 2 — 측정측: FID 게이트에서 image 제외 (4파일)

**문제**(렌즈 2·인용 11/11 실증): 작업 C로 게이트가 켜져 image=area 거짓 FAIL이 15차에 실현. "image 측정 안 함"을 실현하려면 게이트가 image를 무시하게 해야 한다(= 잘못된 측정을 *제거*).

**처방**(image를 L1/L2 게이트 비교에서 *완전 제외*·총수 비교 안 함 → 4결함 무관·positive-control 13케이스 실증 회귀 0):

1. **`workspace/eval/tools/compare_layout.dart`**(eval 단일·미러 불요):
   - **L1**(`:58-59`): `_areas(…).map(role).toList()`에 `.where((x) => x != 'image')` (양쪽 대칭).
   - **L2**(`_flattenSlots` 단일 emit point `:150`): `out.add(s['type'])`를 `else if (s['type'] != 'image') out.add(...)`로 — group 재귀·block·repeat unit 전 경로의 유일 방출구라 여기 한 점이면 L2 전체에서 image 제거(렌즈 2가 emit 전수로 유일성 확증). L1·L2 *둘 다* 필요.
   - **L3 불변**(약신호·게이트 무관·image slot ⚠ 리포트는 육안 재료).
   - **헤더 doc**(`:7` "L1: areas role 시퀀스") 끝에 "(image 제외·위치=§9·L4 육안)" 1구 — doc-drift 방지.
2. **`workspace/eval/tools/layout-ir-schema.md`**(동결 — **노드 트리 불변**): image role/src/alt(§1·`:26·28·38`)는 **그대로**(extractor가 image area 계속 방출 = `ref.json` SoT 보존·`fid-gate.sh:56` ref-layout.json 덤프는 **A1 폴백 경로 한정**·정상 게이트엔 ref.json에 image 잔존). §3 평탄화에 *주석 1줄*: "L1/L2 게이트 비교 시 image 제외(위치=생성측 §9·측정=L4 육안). extractor는 image area 계속 방출."
3. **`workspace/eval/rubric/RUBRIC.md` §H**(`:142` FID-L1 행): **PASS 영역 목록 + FAIL 신호 컬럼 *양쪽*** image 제거 → "appbar/section/bottomnav". `:145` L4 행에 "이미지 존재·위치=육안" 추가. `:147` FID 주의에 "⑥ image 존재/위치는 L4 육안(생성측 §9·fetch_images 담보)·L1/L2 비측정" 추가. **historical 라인**(`:137` "8차 image 누락 통과") = "(현재는 L4 육안 이관)" 시점 표지.
4. **`workspace/eval/rubric/EVAL-METHOD.md`**(`:129` FID-L1 PASS 행): "(image·bottomnav 포함)" → "(bottomnav 포함·image는 L4 육안 이관)". `:162` 시각 충실도 단락에 image=L4 이관 1줄. **잔존 image 단언 전수 scrub**(렌즈 3 지목: `:18·98·125·180·182·190·228·232` 등 — plan에서 grep로 전수 확인 후 image를 FID-L1에서 빼고 L4 이관·historical은 시점 표지).

**trade-off(명시·수용)**: image를 게이트에서 빼면 "이미지 통째 빠짐"(codex 0개)도 자동 미검출(육안 이관). measure-first 빈틈이나 **사용자 확정으로 약신호도 안 둔다** — fetch_images 안정화(별도 라운드)가 진짜 해결(efficacy 자료조사 `:45` "image L1 누락은 14차 거짓 PASS 2/3 자동 차단 신호"를 버리는 것·정직 수용).

**positive-control 재검(선결·상주화)**: ⓐ 섹션 내부 image → PASS ⓑ image 위치 차이(area↔slot) → PASS — **이 둘을 run.sh 회귀 케이스로 상주 추가**(일회성 재검 아님·미래 회귀 방지). ⓒ 기존 7케이스(특히 "L1 영역누락"=image+bottomnav 제거 → **bottomnav로 여전히 정탐 exit 2**·실증) 회귀 0. image-only 정탐 케이스는 육안 이관으로 moot(무의미해질 뿐 vacuous 아님). **통과해야 투입.**

## 5. 측정 = 육안 명문 (RUBRIC)

이미지 충실도(존재·위치·정체)는 **FID-L4 = 사용자 육안**으로 명문화한다(미관·픽셀·아이콘 심볼과 동렬). 도구 비측정. 생성측 §9가 제자리에 그리도록 유도하고, 최종 판정은 사람 눈.
