## 1. 렌즈별 순위 (각자 독립 산출)

| 렌즈 | 순위 | top-2 | 핵심 논리 |
|---|---|---|---|
| 컨텍스트 경제 | F > G ≈ E > A > D > C > B | F·G | 명세 재독(=호출 수)이 지배 비용. 재방문(앞 호출 파일을 되읽고 재수정)이 제2 비용 — C가 재방문 챔피언(S1 6회), B는 같은 재방문 0을 2배 호출로 구매 |
| 실패·복구 | A > C ≈ E > G > D > B > F | A·C(조건부) | "발견 층과 수정 층이 같은 호출 안에 있는가"가 최대 변별(F2 설계 부정합). 계약 의미 오류(F1)는 **분할로 못 잡음**(전 후보 G2 발견) — 레버는 눈 확인 리듬뿐 |
| 품질·규율 | C > B > G > A > E > F > D | C(신규)·B/G(수정) | 재방문 = drift 최대 부패 벡터(A는 S2에서 같은 VM 3회 재진입). 빈혈 차단은 "Model 작성 시 view 과업이 컨텍스트에 없는" B·G가 최강(순서 방어보다 강함). **호출당 신규 ~8파일/1.2k줄 임계** 초과 시 후반부 dropout(수정 모드는 ~6파일) |

## 3. 유력 2안

### 안 1 — 규모 적응형 G-척추 (가로 철학)

- **기본 = G**: 호출 1 = 골격+domain+infra+application 전체(Model), 호출 2 = presentation+배선(View).
- **소형 축퇴**: 전체 신규+수정 ≤ ~7파일(수정 모드 대부분)이면 1호출(F)로 합침.
- **대형 세분**: Model 호출이 임계(~8파일/1.2k줄) 초과하면 애그리거트·계층 단위로 추가 분할.
- **선택적 tracer**: 계약이 가정 기반(G1 승격 케이스)이거나 위험 행위가 있으면 종단 1줄기 슬라이스를 선행.
- 강점: 완전 결정성(분할이 규모 함수 — LLM 판단 0), 빈혈 컨텍스트 차단, 재방문 0, green이 Model/View 규율 경계와 일치, 호출 2회의 경제성.
- 약점: 행위 dropout이 경계에 집중(Model 표면 과소 건설 — View 작성 때 발견) → 보완: 호출 1 내부 순서 "공개 표면(시그니처·State 모양) 먼저 → analyze → 본문", discipline 감사를 Model 완성 직후 배치.

### 안 2 — 행위 세로 A + 결정적 묶기 (세로 철학, dddjango 연속)

- **기본 = A**: 슬라이스 = 행위 1개 종단, 내부 bottom-up.
- **결정적 묶기(전제 조건)**: "같은 파일을 쓰는 행위는 같은 슬라이스" — 파일 교집합 기반 기계 병합(LLM 재량 제거). S2의 토글·롤백·표시가 같은 VM이면 자동 1묶음(사실상 화면 단위로 수렴).
- **위험 우선 정렬(선택)**: 첫 슬라이스를 "조회 최소 경로"가 아니라 "하층 관통 + 계약 위험 최대 행위"로 — tracer 가치 흡수(C′).
- 강점: F2(설계 부정합) 같은-호출 발견·1호출 반송(실패·복구 최대 변별 항목 1위), 행위↔슬라이스↔task 1:1 추적, dddjango 의미론 연속.
- 약점: 묶기 후에도 남는 재방문(화면 횡단 행위)의 drift, 감사가 매번 전 계층 혼합, 경제 중위.

**갈림의 본질**: 두 안 모두 "작으면 합쳐라"로 수렴하므로 소형·수정 모드에선 동치다. 차이는 **중대형 기능**에서 — 안 1은 "재방문 0·결정성·빈혈 차단"(산출물 안정)을, 안 2는 "부정합 조기 발견·행위 추적"(과정 복원력)을 산다. 명세 품질이 높을수록(우리 설계 단계가 두터울수록) 안 2의 장점은 덜 발동되고 안 1의 장점이 남는다.

## 4. 분할 축과 무관한 수렴 발견 (확정 후보 — 어느 안이든 적용)

1. **소형·수정 모드 1호출 축퇴** 명문화 (3렌즈 수렴).
2. **층별 green 래칫 유지** + **수정 파일은 파일별 green**으로 입도 강화 (실패·복구: 래칫이 F4 변별력의 대부분을 만듦 / 품질: 무기억 에이전트의 자가 복구 조건).
3. **호출 경계를 넘는 타입 스텁 파일 선생성 금지** (재방문 0 후보를 재방문 N으로 전환). 단 **호출 내부**의 "시그니처 먼저 → 본문" 순서는 긴 호출에서 권장.
4. **감사 임계 보완**: "슬라이스 ≥3" 규칙에 "**Model/View 경계 통과 시 1회**"를 추가 — 판정 소유 감사의 최고가치 시점(판정이 결정 가능해지는 첫 순간, presentation 소비 전 교정 비용 최저).
5. **discipline-reviewer 홀리스틱에 "명세 행위 목록 ↔ 코드 실현 대조" 항목** — 가로 계열의 행위 부분 누락(VM 메서드 1개) 백업.
6. **coder 호출 시작 전 git 스냅샷 절차화** — F3(호출 중단) 복구를 "revert 후 동일 입력 재호출"로 기계화(후보 간 격차 자체를 소멸시킴).
7. **F1(계약 의미 오류)은 어떤 분할도 못 잡는다** — 가정 계약(G1 승격) 케이스에 한해 첫 실행 가능 시점의 **선택적 중간 눈 확인**(Coordinator 재량)을 G2 외 1회 허용할지 검토.
8. bottom-up은 전 후보 공통 이득(참조 실재) — 단 세로 계열에선 유일한 빈혈 방어라 절대 유지, 가로 계열에선 보조.
