# FID 게이트(구조 비교) 실효성 자료조사 — 2026-06-22

> **질문**: 현재 FID 게이트(screenProbes + 구조 IR 비교)를 **고치면** 사용자의 "무조건 재현"(시안 요소 가감 0)을 **자동 검증할 실효가 있는가**? 못 잡는 경계는 어디고, 그 경계를 메울 현실적 보완은?
> **트리거**: 14차 라이브런에서 사용자 육안이 "codex 리스트가 시안과 다름"(브로콜리 이미지 누락·condition 텍스트 라벨 추가)을 잡았으나, grep 채점은 "형상 6/6 시안 일치" **거짓 PASS**. 사용자 지시 = "스크린샷 픽셀 비교는 제외(결정론 부적합) + **고치기 전에 고치면 실효 있을지** 자료조사".
> **방법**: 3갈래 ultracode fan-out — ① 내부 실증(14차 산출물을 실제 도구체인 `extract_layout`→`dump_to_ir`→`compare_layout --gate`에 통과) ② 업계 관행 외부 리서치 ③ 보완책 외부+추론. 합성=메인.
> **한계**: N=1(단일 회차 실측)·in-family(분석가 전원 Claude 계열). 인과 단정 금지.

## 0. 핵심 결론 (4줄)

1. **구조 IR 비교 채택(2026-06-19 결정)은 능동적으로 옳았다** — 당시 자료조사가 golden(픽셀)·AI비전을 *논한 뒤 명시적으로 배제*하고 **결정론·무의존·게이트 가능**을 이유로 구조 IR을 택함. 사용자의 "스크린샷 제외"와 정합하며, 업계·학계도 동일 결론(게이트 가능한 건 결정적 측정뿐).
2. **고치면 실효 있다(실측)** — screenProbes만 켜면 14차 3대 결함 중 **2개(브로콜리 누락·bottomnav 누락)는 L1이 자동 FAIL**시킨다. 14차 거짓 PASS의 진짜 원인은 "구조 비교가 틀렸다"가 아니라 **"장치가 6회 연속 안 돌아 grep이 대신했다"**.
3. **그러나 1개(condition 라벨 추가)는 현 로직이 못 잡는다** — `_collapse`(인접 동종 흡수)가 먹어 게이트 통과. + 실측 중 **도구 결함 2개 추가 발견**(image 위치 의존·repeat 접미사 의존) → 고칠 때 함께 치유 필요.
4. **색·텍스트 내용·이미지 정체·간격·비례·실폰트는 구조 IR이 원리적으로 못 본다 → 영구 사용자 육안.** 단 그중 **텍스트 정적라벨·색·아이콘 심볼·이미지 정체는 "내용 확장 IR"로 결정적 회수 가능**(Design2Code LLEM이 학술 검증).

→ **권고: 2층 방어선.** 1층(자동·결정적 게이트) = screenProbes 강제 + 구조 IR을 *내용까지* 확장 + 도구 결함 치유. 2층(반자동) = 라이브런 사용자 캡처 + AI 고정 루브릭으로 색·실폰트·미세 비주얼 보완. **AI 비전 자동 게이트·스크린샷 픽셀 비교는 배제**(비결정).

## 1. 이전 결정 재평가 — 구조 IR 채택은 옳았나 (내부)

`2026-06-19-stitch-fidelity-research.md` 핵심 논거(구조 IR은 *소거법 기본값이 아니라 능동 선택*):

- **채택 사유**: ① IR-first가 LLM 직재현보다 레이아웃 충실도 우위(ScreenCoder Position Alignment 0.840 vs GPT-4o 0.811, 학술). ② **전제검증(research.md:82-95)**: dddart 9개 런 시안 실측 — Stitch HTML은 div-soup 아님(`<header>/<section>/<nav>` 시맨틱·`absolute`=0·`order-`=0 → DOM순서=시각순서). 결론: "layout-ir 결정론 추출은 기술 실현가능성·결정성 高·신규 의존 불필요."
- **golden/스크린샷 배제(research.md:46)**: "golden test = 환경 재현성이지 UI 품질 아님. 폰트·플랫폼·버전 의존. **픽셀 동일 강제는 LLM 산출 평가에 부적합.**"
- **VLM 비전 배제(research.md:18,45)**: "VLM-as-judge는 게이트가 아니라 회귀 신호로만(전문가에 15~20%p 미달). **기능등가 맹점** — 등가 레이아웃을 기각해 dddart 목표와 충돌."

**판정: 채택 근거는 6회차 운영으로도 유효.** 뒤집힌 것은 도구 설계가 아니라 **"게이트가 실제로 돈다"는 운영 가정**(아래 §2).

## 2. 14차 결함 탐지 실증 (screenProbes 켜졌다고 가정·실제 도구 출력)

시안 SoT(`ref-layout.json`, 양 엔진 byte-동일) list 영역 = `[appbar, image(브로콜리), section{repeat[text,icon,text]}, bottomnav]`. 코드 IR을 14차 실제 소스에서 충실 재구성해 실제 `compare_layout.dart --gate` 통과.

### 3대 결함 잡힘/놓침

| 결함 | 엔진 | 잡는 계층 | 자동 게이트 FAIL? | 근거 |
|---|---|---|---|---|
| **브로콜리 이미지 누락** | codex | **L1** | ✅ **잡음**(exit 2) | image area 0 → `누락=[image]` (스키마가 `<img>`/`Image`를 image 영역으로 모델·`layout-ir-schema.md:53`) |
| **condition 텍스트 라벨 추가** | codex | L3만(약신호) | ❌ **놓침** | `_collapse`(`compare_layout.dart:136-142`)가 인접 text 흡수: `[text,text,icon,…]`→`[text,icon,…]` = 시안과 동일 → L2 PASS. L3은 게이트 아님(→사용자 눈) |
| **bottomnav 누락** | 양 엔진 | **L1** | ✅ **잡음**(exit 2) | nav area 0 → `누락=[bottomnav]` |

> **함의**: screenProbes만 켰다면 codex는 L1(image·nav 누락)으로 **픽스처 FAIL** 도장이 찍혔을 것 = 14차 거짓 PASS의 2/3가 자동 차단됐다. 못 잡는 1/3(condition 추가)이 `_collapse`의 구조적 사각.

### 추가 발견 — 도구 결함 2개 (고칠 때 함께 치유)

- **결함 #1 (image 위치 의존성·거짓 FAIL 위험)**: claude는 브로콜리를 *실제로 그렸는데도*(`weekly_forecast_section.dart:33`) 로고를 `*Section` 위젯 **내부**에 넣어, `dump_to_ir._collectAreas`가 Section 경계에서 멈춰 내부 Image를 top-level image area로 승격 못 함 → **L1 image 누락 + L2 image 추가로 거짓 FAIL**. 시안은 `<img>`가 `<section>`의 *형제*. → 시안이 로고를 섹션 밖에 두는데 코드가 섹션 안에 중첩(합리적 선택)하면 옳게 그려도 FAIL.
- **결함 #2 (repeat-group이 위젯명 접미사 의존)**: `_isUnit`(`dump_to_ir.dart:64`)이 `InkWell`/`GestureDetector`/`*Card`/`*Tile`만 반복단위로 인정. 14차엔 타일이 우연히 `InkWell`/`GestureDetector` 래퍼를 가져 통과했으나, 순수 `Container` 타일이면 전 타일이 평탄 슬롯열로 합쳐져 거짓 FAIL.

## 3. 구조 IR이 원리적으로 못 보는 것 (게이트 완벽히 고쳐도 영구 A1)

스키마 §0 불변(`layout-ir-schema.md:11-16`)이 명시 배제: "의미 라벨 없이 시각 타입+배치+순서만·픽셀 값 미포함."

| 차원 | 14차 실예 |
|---|---|
| 이미지 정체 | 브로콜리든 고양이든 image 1개면 통과 |
| 텍스트 내용 | "맑음" vs "흐림", 날짜 포맷 오류 불검출 |
| 색 | condition별 색(sunny=amber) 틀려도 불검출 |
| 폰트 크기·간격·비례 | 80px가 16px여도, 카드 높이·여백 틀려도 불검출 |
| 동종 인접 개수 | text 2 vs 3 불검출(= condition 결함의 근본) |

`EVAL-METHOD.md:162`가 이미 못박음: "미관(L4)은 렌더 미관측이라 사용자 육안 대조에 위임." → **색·정체·내용·비례는 구조 게이트로 영구 미포착.** (단 일부는 §5 내용확장으로 회수.)

## 4. 업계 관행 (외부 리서치)

- **"100% 동일 재현"을 단일 점수로 자동 채점하는 검증된 업계 표준은 없다.** 픽셀=비결정(폰트/렌더러/플랫폼), AI=비결정, 트리=시각 맹점. 모두 단독 불완전.
- **결정론 파이프라인에 맞는 건 결정적 측정뿐**: 픽셀/SSIM, 트리(TED·TreeBLEU), **요소/블록 매칭(LLEM·Block-Match)**. CLIP·LLM-judge는 비결정이라 게이트 부적합(진단 보조만).
- **Design2Code(Stanford)의 LLEM** = 매칭 요소의 **존재·텍스트·위치·색을 category별 recall**로. "요소 가감 없는 재현"에 개념적으로 가장 근접 — **"가감 0" = 요소 존재 recall/precision = 1.0**으로 자연스럽게 메트릭화.
- **상용 디자인-투-코드 도구(Locofy·Anima·Stitch·builder.io) 중 시안 충실도를 정량 자동 검증한다고 공개·독립검증된 곳은 없다** — 전부 자체보고 + 인간 사후수정 전제("AI gets ~75% of the way"). → dddart가 결정적 충실도 게이트를 구축하면 **차별화 지점**.

출처: Design2Code(arXiv 2403.03163) · matchesGoldenFile/Alchemist(api.flutter.dev·pub.dev) · BackstopJS/Percy/Chromatic/Applitools 비교 · MLLM-as-UI-Judge(arXiv 2510.08783) · AX-tree 비교(AXeRay ICSE'21).

- **내용 확장 IR의 결정성 함정 = 동적 데이터 텍스트**(날짜/카운트 등 런타임 가변). 처방: IR 어휘에 `bound=true` 표식 후 그 슬롯은 **내용 비교 제외, 존재+type+포맷만** 검증. 정적 라벨(버튼·헤더·카피)만 내용 일치.

## 6. 종합 권고 — 2층 방어선

**1층 (자동·결정적·green 게이트):**
1. **screenProbes 강제** — `coder.md` 필수 산출로 이전(6회 미해결 근원). 이것만으로 14차 결함 2/3 자동 포착.
2. **구조 IR을 내용까지 확장** — 슬롯에 텍스트 정적라벨·색·아이콘 심볼명·이미지 정체(src/alt) 부착 후 **결정적 양방향 set-diff**(가감 검출). 동적 데이터는 `bound` 표식 제외. → condition 라벨 추가 같은 가감 포착.
3. **도구 결함 2개 치유** — image area 위치 정규화(Section 내부 Image도 승격 or section-slot 인정) + repeat-group 접미사 의존 완화. (선치유 안 하면 거짓 FAIL.)
4. (선택) `_collapse` 보정 — "인접 동종 흡수하되 개수 차가 임계 이상이면 신호". 단 8차 false-regression과 트레이드오프 → positive-control 재검 필요.
5. 보조: 시맨틱 트리 덤프(텍스트 라벨·역할 대조).

**2층 (반자동·사용자 수용 — 1층이 원리적으로 못 보는 것):**
- 라이브런 때 사용자가 화면별 스크린샷 제공 → AI(조정자)가 **자유 인상평이 아니라 고정 루브릭**(시안 요소 존재 체크리스트·텍스트·색군·이미지 정체)으로 판정. 사람이 루프에 있어 환각 흡수. = 현재 육안 피드백의 구조화. **색·실폰트·미세 비주얼** 담당.

**배제:** AI 비전 자동 게이트(비결정·유사 UI 둔감), 스크린샷 픽셀 비교(렌더 엔진 상이) — **사용자 "스크린샷 제외" 결정이 자료로 확증됨.**

## 7. "고치면 실효 있나" 최종 판정

**있다 — 단 조건부.** 단순히 screenProbes만 켜는 것(6회 미해결 봉합)으로도 14차 결함의 2/3가 자동 차단되어 거짓 PASS가 안 났을 것이다. 나머지(가감 검출)는 **내용 확장 IR**이 결정적으로 회수하며 이는 학술(LLEM)·업계 수렴 방향이다. 색·실폰트·미세 비주얼은 어떤 결정적 게이트로도 못 보므로 **2층 반자동(사용자 캡처+AI 루브릭)**이 영구 보완. **장치를 버릴 이유는 없고, 켜고+내용확장+버그치유로 살리는 게 비용 대비 실효가 가장 높다.**
